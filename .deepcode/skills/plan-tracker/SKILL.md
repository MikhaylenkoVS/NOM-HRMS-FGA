# Plan Tracker

Отслеживает планы проекта: сохраняет в `docs/plans/`, выводит дневную сводку и просроченные задачи.

## Правила

### 1. Сохранение планов

- **Всегда** сохраняй любой утверждённый план как `docs/plans/<имя-плана>.md`.
- Формат: Markdown с чекбоксами `[ ]` / `[x]` для каждого пункта.
- В начале файла — мета-блок: `> **Создан:** YYYY-MM-DD | **Статус:** 🔴/🟢 | **Дедлайн:** YYYY-MM-DD`.
- При изменении статуса задач — обновляй файл плана (отмечай `[x]`).

### 2. Вывод плана на день

- **В начале каждого разговора** (но не чаще 2 раз в день) выводи:
  1. Заголовок `## 📋 План на сегодня (<день недели>, DD.MM.YYYY>)`
  2. Актуальные задачи на сегодня из всех активных планов в `docs/plans/`.
  3. Блок `### ⚠️ Просрочено` с задачами, у которых дедлайн был вчера или раньше и они не отмечены `[x]`.
  4. Краткую сводку: `Выполнено: X/Y, осталось: Z`.
- Если разговор уже второй за день — не выводи.

### 3. Обновление статуса

- При любом изменении состояния задач — обновляй `docs/plans/*.md`.
- При закрытии всех задач плана — меняй статус на `🟢 завершён`.

### 4. Интеграция с GitHub Issues

- Если в плане упомянуты GitHub issues (например, `#27`), при закрытии задачи на GitHub — обновляй статус в плане.
- При запросе пользователя «какой статус» — проверяй GitHub API и локальный план, выводи сводку.

## Формат плана

```markdown
# Название плана

> **Создан:** YYYY-MM-DD | **Статус:** 🔴 в работе | **Дедлайн:** YYYY-MM-DD

## Сводка
...

## День 1 — День недели, DD.MM
| # | Задача | Статус |
|---|--------|--------|
| ... | ... | [ ]/[x] |

## День 2 — ...
...
```

## Проверка дублирования

- При сохранении нового плана проверять, нет ли уже плана с таким именем в `docs/plans/`.
- Если есть — обновить существующий, а не создавать дубликат.
